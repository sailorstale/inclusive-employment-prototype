# Аудит понятности глазами роли

Проход по страницам сайта от лица его аудитории: где текст непонятен, какие вопросы возникают у пользователя, что предложить. Роль для каждой страницы берётся из трека (раздела): `/companies` → работодатель, `/ngo` → куратор НКО, `/jobseekers` → соискатель с инвалидностью, общие страницы → новый посетитель.

**Как это устроено.** По каждой странице агент-читатель входит в роль и читает страницу целиком, отмечая затыки; затем агент-скептик отсеивает придирки и додумки, сверяясь с реальным текстом. Флоу переиспользуемый: `.claude/workflows/persona-clarity-audit.js`. Запуск на трек: `Workflow(name: persona-clarity-audit, args: "companies" | "ngo" | "jobseekers" | "shared")`, на страницу — `args: "/companies/legal/quotas"`.

## Покрытие

Прогнано **28 из 40** содержательных страниц. Не хватает **9** — на них сорвался шаг проверки из-за месячного лимита расходов аккаунта (не лимит по токенам). Чтобы догнать — поднять лимит на claude.ai (Settings → Usage) и прогнать оставшееся:

```
Workflow(name: persona-clarity-audit,
  args: ["/yandex-jobs", "/ngo", "/ngo/how", "/ngo/support", "/ngo/candidates",
         "/ngo/scale", "/companies/legal/faq", "/companies/hire/step-6", "/companies/team"])
```

## Три вывода, общие для всего сайта

1. **«Витрина одного работодателя».** Почти везде контент завязан на Яндекс без честной оговорки о природе ресурса — недоверчивый пользователь читает это как рекламу, а не независимую помощь. Особенно бьёт по соискателю.
2. **Термины и юрфакты без пояснения «на пальцах».** Квота, ИПРА, МСЭ, ГПХ, ЦЗН, СФР, РЖЯ, «сопровождаемое трудоустройство» — вводятся непоследовательно; часть в тултипах, которые торопливый читатель не открывает.
3. **Самое нужное спрятано.** Ответ на главный страх роли (штрафы и суммы — у работодателя; права и раскрытие диагноза — у соискателя; методика и источники — у НКО) либо в футере длинной страницы, либо на внешнем сайте, либо отсутствует.
