# Деплой прототипа на Railway

Прототип = фронт (Vite) + мини-сервер (Express, хранит правки и комментарии в
файлах). Цель: опубликовать на Railway, чтобы правки были общими и не пропадали.

В репозитории уже готово: `Dockerfile` (сборка), `railway.json` (настройки),
сервер читает `PORT` / `DATA_DIR` из окружения.

> **Пароля нет — доступ открыт по ссылке.** Любой, кто знает адрес, может читать,
> менять и удалять правки и комментарии; удаление необратимо. Единственная
> страховка — бэкапы (см. «После публикации»).

---

## Что задать в Railway (важное)

| Переменная | Значение | Зачем |
|---|---|---|
| `DATA_DIR` | `/data` | папка тома, где лежат правки/комментарии |
| `PORT` | *(не задавать)* | Railway проставит сам |

**Том (Volume)** монтируется в **`/data`** — без него данные сотрутся при
следующем передеплое (это критично, риск C1).

---

## Вариант A — без GitHub, через Railway CLI (проще)

1. Установить CLI: `npm i -g @railway/cli`, затем `railway login`.
2. Из папки `prototype/`: `railway init` (создать проект).
3. В дашборде проекта (railway.app): **Variables** → добавить `DATA_DIR=/data`.
4. **Settings → Volumes** → New Volume → Mount path `/data`.
5. Деплой: из `prototype/` выполнить `railway up`. Railway соберёт образ по `Dockerfile` и запустит.
6. **Settings → Networking → Generate Domain** — получишь адрес `…up.railway.app`.
7. Обновлять сайт после правок кода: снова `railway up` из `prototype/`.

## Вариант B — авто-деплой при пуше (через GitHub)

1. Создать репозиторий на GitHub и запушить проект (попроси меня — подскажу команды).
2. Railway → **New Project → Deploy from GitHub repo** → выбрать репозиторий.
3. **Settings → Root Directory** → `prototype` (код прототипа лежит там).
4. **Variables**: `DATA_DIR=/data`.
5. **Settings → Volumes** → Mount path `/data`.
6. Railway соберёт по `Dockerfile` автоматически. Каждый новый пуш → авто-передеплой, данные на томе сохраняются.

---

## Свой домен

**Settings → Networking → Custom Domain** → ввести домен → Railway покажет
`CNAME`, который надо прописать у регистратора домена. После проверки — сайт на
твоём адресе.

## После публикации

- Открыть сайт (пароль не спрашивается).
- Включить «Редактор», поправить тексты; разработчик и клиент работают на
  странице **«Изменения»**.
- **Бэкап:** периодически жми «Скачать бэкап» на `/changes` (выгрузка всех
  правок и комментариев в JSON) — страховка на случай проблем с томом.

> Примечание: GitHub Pages-деплой (`.github/workflows/deploy.yml`) — отдельная
> витрина без сервера; там правки в общий стор не сохраняются. «Боевой» общий
> редактор — это Railway-версия.
