# Канон консистентности контента

> Источник правды по единообразию текстов и оформления. Ведётся по результатам
> аудита консистентности (см. артефакт-отчёт). Решения принимает дизайнер;
> здесь фиксируется принятое. **Главный принцип — сохранность контента:** канон
> не сливает вещи, различие которых несёт смысл.

Обновлено: 2026-07-05.

## Терминология

### наём (не «найм»)
Каноническое написание — **«наём»** (через «ё»), во всех позициях, включая
навигацию. Раздел меню — **«Наём по шагам»**. Падежи «найма/найме/найму/наймом»
не меняются (это правильные формы). — *Внедрено 05.07 (коммит с nav.ts).*

### соискатель / кандидат  ⚠️ политика, НЕ автозамена
Слова означают **разное** и не взаимозаменяемы:
- **соискатель** — человек, ищущий работу **вообще** (вне привязки к конкретной
  вакансии): в разделах про поиск работы, профессии, истории, общие портреты.
- **кандидат** — конкретный претендент **на вакансию / в процессе найма**: в
  разделах про отбор, собеседование, оформление, работу рекрутера с человеком.

Правило применяется **точечно по контексту**, массовая замена запрещена (сломает
смысл). Выверка сайта по этому правилу — отдельный аккуратный проход (955 вхождений).
Уже поправлен явный выброс: «портрет кандидата» → «портрет соискателя» (NgoCandidates).

## Вводная врезка «что вы узнаете»
Единый вид на всех страницах:
```
<Callout variant="briefing" title="В этом разделе вы узнаете">
```
- вариант — **briefing** (не info);
- слово — **«разделе»** (не «модуле»/«материале»);
- **без двоеточия** в заголовке.
— *Внедрено 05.07 (19 из 21; CompaniesHow/CompaniesStart — догнать; Step1/Step4
голым текстом — обернуть во врезку, проход A4).*

## Итоговый блок
- Заголовок — **«Подведём итоги»** (мн. ч.), не «Подведём итог». *(внедрено)*
- **«Заключение» ≠ «Подведём итоги»** — разные слоты: «итоги» = конспект сути
  страницы; «Заключение» = финальное напутствие читателю. **Не сливать.**

## Вводная секция страницы
Заголовок — **«Вступление»** (не «Введение»). *(внедрено)*

## Блок обратной связи
Заголовок — **«Остались вопросы? Есть пожелания?»** *(внедрено)*

## Заголовки примеров
Формат — **«Пример. Суть»**: слово «Пример» + точка + о чём пример, без
завершающей точки. Образец: «Пример. Как зарплата влияет на итоговый доход».
Безликие «Пример» / «Давайте рассмотрим на примере» / «Это один из примеров»
не используются. Нумерованные серии («Пример 1. …») — уже в этом формате.
— *Внедрено 05.07 (все 13).*

## Интерактивные блоки-самопроверки: «Проверьте себя», не «Квиз»
Единый ярлык — **«Проверьте себя»** (слово «квиз» — след учебного курса, см.
редполитику; в текстах сайта не используется). Заголовок с уточнением — через
двоеточие: «Проверьте себя: кто берёт инициативу — работодатель или НКО».
Подпись блока разбора внутри — **«Ответ»**. — *Внедрено 05.07: 2 заголовка
секций (CompaniesHow/NgoHow), 2 подводки терминологического блока
(TeamPage/NgoEthics), 46 подписей разбора + дефолт компонента.*

**Заголовок «Проверьте себя» стоит над каждым блоком самопроверки.** Если
секция целиком самопроверка — её заголовок в формате «Проверьте себя: …»;
если самопроверка в хвосте тематической секции — вложенный h3 «Проверьте себя».
— *Внедрено 05.07: терминология и ситуации (TeamPage/NgoEthics), ситуации
собеседования (Step4), кейсы вакансий (Step1), «с кем начинать» и 2 ситуации
(NgoStart), обёртка 8 ситуаций (Step5).*
**Исключение:** одиночные интерактивные разборы, вплетённые в повествование
(алгоритм возражений NgoEmployers, сквозной кейс интервью NgoCandidates), —
без ярлыка: заголовок разрывал бы связный текст.

## Варианты ответов в блоках «Проверьте себя»
С **заглавной буквы**, **без завершающих «;» и «.»** у коротких вариантов
(термины, фразы-фрагменты). Варианты — полные предложения (сценарные блоки)
остаются предложениями с точкой. — *Внедрено 05.07 (53 варианта, 4 файла).*

## Служебные якоря (anchor)
Схема итоговых секций — `<префикс>-summary` (не голый `summary`, не транслит
`itogi`). Правлено точечно (Step6 `itogi` → `step6-summary`).

## Порядок блоков в конце страницы
**StepNav («предыдущий/следующий») → затем RelatedLinks** («Читайте также»).
*(внедрено 05.07 на Step1/Step3/Step4.)*

## Нумерация шагов
На NgoCandidates два ряда — **«Этап 1–6»** (профориентация) и **«Шаг 1–8»**
(поиск работы) — **оставлены намеренно**: разные слова разводят два процесса.
Не сводить к одному.

## Типографика
- Тире в пояснительных конструкциях — длинное **«—»** (не дефис «-»). *(проход вручную)*
- Номер — **«№ N»** с пробелом. *(внедрено, где найдено)*
- Завершающая пунктуация внутри одного списка — единая. *(проход вручную)*
- Короткие заголовки-врезки (Callout) — **без завершающей точки**. *(внедрено 05.07.)*

## Семантический компонент «Пример»  ✅ решено 2026-07-05
Пример — это иллюстрация/кейс/образец формулировки, **НЕ цитата**. Компонент —
**`<Example>`** — тонкая обёртка над **`Callout variant="highlight"` с заголовком
«Пример»** (это и есть канонический вид примера на сайте: брендовая врезка со
звёздочкой; десятки таких уже стоят вручную, напр. Льготы «Пример. Как зарплата
влияет на доход», Кандидаты «Пример 1…»). Обёртка фиксирует вид — по образцу
`Warning` над `Callout · warning`. Заголовок по умолчанию «Пример», можно
опустить (`title={null}`) для контрпримеров или уже введённых текстом.
Часть примеров через `Blockquote` выглядели как чужие цитаты с пустым автором
(аватар · имя · должность · логотип) — это и чинится переводом в `<Example>`. Переведены все 15 таких блоков
(Шаг 2, Шаг 4 ×3, НКО-Кандидаты ×5, НКО-Работодатели ×6). Ярлык «Пример»
показываем, когда пример стоит самостоятельно; скрываем (`label={null}`), когда
текст уже вводит пример словами. **Blockquote остаётся только для настоящих
именных цитат.**

## Ждёт решения дизайнера (жёлтое/структурное)
- **Компоненты одного смысла** («Миф/Факт», «Практическое задание») —
  решаются **в потоке унификации D** (семантические компоненты).
- «Важно»: различить предупреждение (`Warning`) и нейтральный акцент (`info`) —
  не автозамена (красное).
- Формат итога в НКО: секция vs `Callout highlight`.
- «Итоговые рекомендации» → свести к «Подведём итоги» или оставить подвидом.
- Структурные пробелы (достраивать скелет или принять как есть): итог не на всех
  правовых подстраницах; блок обратной связи не на всех шагах; StepNav не на всех
  страницах НКО; Step5 без итога/задания.
- Дубль h3 «Как составить описание вакансии» на Step4 — переименовать второй.
