# Канон консистентности контента

> Источник правды по единообразию текстов и оформления. Ведётся по результатам
> аудита консистентности (см. артефакт-отчёт). Решения принимает дизайнер;
> здесь фиксируется принятое. **Главный принцип — сохранность контента:** канон
> не сливает вещи, различие которых несёт смысл.

## Терминология

### наём (не «найм»)
Каноническое написание — **«наём»** (через «ё»), во всех позициях, включая
навигацию. Раздел меню — **«Наём по шагам»**. Падежи «найма/найме/найму/наймом»
не меняются (это правильные формы). — *Внедрено 05.07 (коммит с nav.ts).*

### соискатель / кандидат  ⚠️ политика, НЕ автозамена
Слова означают **разное** и не взаимозаменяемы:
- **соискатель** — человек, ищущий работу **вообще** (вне привязки к конкретной
  вакансии): в разделах про поиск работы, профессии, истории, общие портреты.
- **кандидат** — конкретный претендент **на вакансию / в процессе найма**: в
  разделах про отбор, собеседование, оформление, работу рекрутера с человеком.

Правило применяется **точечно по контексту**, массовая замена запрещена (сломает
смысл). Выверка сайта по этому правилу — отдельный аккуратный проход (955 вхождений).
Уже поправлен явный выброс: «портрет кандидата» → «портрет соискателя» (NgoCandidates).

## Вводная врезка «что вы узнаете»
Единый вид на всех страницах:
```
<Callout variant="briefing" title="В этом разделе вы узнаете">
```
- вариант — **briefing** (не info);
- слово — **«разделе»** (не «модуле»/«материале»);
- **без двоеточия** в заголовке.
— *Внедрено 05.07 (19 из 21; CompaniesHow/CompaniesStart — догнать; Step1/Step4
голым текстом — обернуть во врезку, проход A4).*

## Итоговый блок
- Заголовок — **«Подведём итоги»** (мн. ч.), не «Подведём итог». *(внедрено)*
- **«Заключение» ≠ «Подведём итоги»** — разные слоты: «итоги» = конспект сути
  страницы; «Заключение» = финальное напутствие читателю. **Не сливать.**

## Вводная секция страницы
Заголовок — **«Вступление»** (не «Введение»). *(внедрено)*

## Блок обратной связи
Заголовок — **«Остались вопросы? Есть пожелания?»** *(внедрено)*

## Служебные якоря (anchor)
Схема итоговых секций — `<префикс>-summary` (не голый `summary`, не транслит
`itogi`). Правлено точечно (Step6 `itogi` → `step6-summary`).

## Типографика
- Тире в пояснительных конструкциях — длинное **«—»** (не дефис «-»). *(проход вручную)*
- Номер — **«№ N»** с пробелом. *(внедрено, где найдено)*
- Завершающая пунктуация внутри одного списка — единая. *(проход вручную)*

## Ждёт решения дизайнера (жёлтое)
- Вариант компонента для «Пример», «Миф/Факт», «Практическое задание» — в рамках
  потока унификации D (семантические компоненты).
- «Важно»: различить предупреждение (`Warning`) и нейтральный акцент (`info`) —
  не автозамена.
- Формат итога в НКО: секция vs `Callout highlight`.
- Наличие итога на всех правовых подстраницах; блок обратной связи на всех шагах;
  StepNav на всех страницах НКО; Step5 (нет итога/задания); порядок хвоста
  StepNav↔RelatedLinks; точка в коротких заголовках-врезках; дубль h3 на Step4;
  нумерация «Шаг N»/«Этап N» на NgoCandidates.
